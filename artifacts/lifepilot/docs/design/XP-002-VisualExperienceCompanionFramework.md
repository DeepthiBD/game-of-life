# XP-002 — Visual Experience & Companion Framework

**Version**: 1.0  
**Status**: Active — Binding on all modules, current and future  
**Authority**: Overrides generic SaaS UI conventions  
**Governs**: Visual Experience · Companion System · Avatar Guidelines · Conversation Design · Journey Design · Screen Atmosphere · Visual Storytelling · Child Engagement · Emotional Design · Animation · Illustration · Badge Philosophy

---

## Core Experience Principle

> A child should feel: **"I am on my own adventure."**  
> NOT: **"I am using educational software."**

---

## Experience Philosophy

LifePilot should create:  
**Wonder · Discovery · Exploration · Curiosity · Confidence · Hope · Optimism · Belonging · Future Thinking**

---

## Companion-First Experience

LifePilot should feel **conversational**. Children should feel **accompanied**. Children should never feel **managed**.

### Forbidden UI Patterns
Forms · Spreadsheets · Tables · Administrative UI · Long questionnaires · Corporate dashboards · ERP-style layouts

### Preferred Card Formats
| Card Type | Use For |
|---|---|
| Conversation Card | Companion dialogue, guided moments |
| Story Card | Narrative moments, module introductions |
| Choice Card | Decisions, crossroads, life choices |
| Journey Card | Progress, milestones, flight plans |
| Discovery Card | Career worlds, new topics |
| Mission Card | Today's adventure, daily goals |
| Reflection Card | Flight log, mood, journal |

## Companion System

### Companion Naming Principles
Companions should feel friendly, warm, memorable, easy to pronounce, suitable across India, suitable globally.  
**Companions are friends. Not instructors. Not authority figures. Not teachers.**

## Visual Immersion Principle

> **LifePilot should feel like worlds. Not pages.**

Every major screen must contain:
**Atmosphere · Illustration · Companion Presence · Journey Element · Primary Action · Visual Storytelling · Discovery Opportunity**

---

## Screen Structure Requirements

Every major screen must include all seven elements:

| Element | Description |
|---|---|
| **Companion** | Companion card or avatar visible and contextually present |
| **Illustration** | Atmospheric illustration, world element, or journey visual |
| **Journey Context** | Module-appropriate metaphor (flight path, world, crossroads, horizon) |
| **Primary Action** | One clear, adventure-toned CTA |
| **Exploration Opportunity** | A way to discover more — another world, path, or choice |
| **Progress Context** | XP, milestone, or journey progress — visible but not dominant |
| **Emotional Context** | Hopeful, encouraging tone in all copy and illustration |

## Experience Themes

### Primary — Sky Adventure
**Feelings**: Freedom · Possibility · Exploration · Growth · Adventure  
**Visual elements**: Clouds · Flight paths · Airplanes · Sunlight · Sky gradients · Horizon lines  
**Background**: `#DFF3FF` → `#F7FBFF` → `#FFF8F0` (top to bottom)

### Secondary — Journey Map
**Feelings**: Discovery · Progress · Adventure  
**Visual elements**: Trails · Paths · Destinations · Landmarks · Exploration routes · Journey progress

### Special — Future Horizon *(Future Me only)*
**Feelings**: Hope · Wonder · Possibility · Optimism  
**Visual elements**: Sunrise · Warm glow · Floating messages · Journey trail  
**Background**: Warm sunrise gradient, soft glow, possibility and distance

---

## Background Principles

**Never use flat white screens.**

Use: Sky gradients · Cloud layers · Soft horizons · Journey paths · Subtle illustrations · Atmospheric elements

Backgrounds must **support storytelling**. Every background is a world, not a canvas.

**Dark Mode**: Night Sky (`#0F172A`). Use stars, constellations, moonlight, soft gradients. Must still feel adventurous — never plain black.

---

## Module Experience Standards

### Cockpit (Home Base)
> "Cockpit is home base. NOT dashboard."

Required sections:
1. Welcome Captain
2. Message From Future You (companion card)
3. Today's Adventure (mission card)
4. Explore New Worlds (module grid)
5. Journey Progress (XP bar)
6. Upcoming Milestones (badge/milestone card)

Visual storytelling required. Forbidden: KPI dashboards, admin task lists.

### Future Me *(most magical experience)*
Atmosphere: Sunrise · Future Horizon · Warm Glow · Floating Messages · Journey Trail  
Emotional goals: Wonder · Optimism · Hope  
Forbidden: Metrics, scores, progress percentages, evaluation

### Career Explorer
> "Careers are worlds. Not database entries."

Career worlds: 🚀 Space · 💻 Technology · 🏥 Health · ⚖️ Law · 🎨 Design · 📈 Entrepreneurship  
Children should feel they are **exploring**, not searching.  
Companion: 🧭 Tara guides exploration.

### Money Quest
> "Money is an adventure."

Card types: Decision Cards · Choice Paths · Story Moments · Exploration Routes · Trade-off Challenges  
Forbidden: Spreadsheets · Financial dashboards · Accounting interfaces

### Flight Log
> "Reflection should feel safe, personal, calm, supportive."

Emotional goals: Safe · Personal · Calm · Supportive · Reflective  
Forbidden: Homework feel · Assessment · Evaluation · Grading

## Colour Principles

Colours should feel: **Safe · Friendly · Warm · Hopeful · Adventurous · Optimistic**

Avoid: Harsh black · Enterprise palettes · Neon colours · Aggressive reds · Pure `#000000` text

Prefer deep navy (`#17324D`) and soft blue-grey (`#5F7D95`) for all text.

---

## Typography Principles

**Friendly · Readable · Human · Warm · Accessible**

Avoid: Corporate typography · Condensed typography · Technical appearance

Never use pure black. All body text minimum 16px, line-height 1.6.

---

## Animation Principles

Animations should feel: **Natural · Gentle · Calm · Encouraging**

### Allowed
Cloud Drift · Card Float · Path Reveal · Message Arrival · Badge Glow · Journey Expansion

### Forbidden
Flashing · Casino effects · Aggressive motion · Attention traps · Manipulative engagement loops · Auto-advance

All animations must degrade to instant state change when `prefers-reduced-motion: reduce` is active.

---

## Badge Philosophy

### Celebrate
Curiosity · Consistency · Reflection · Kindness · Exploration · Responsibility · Leadership · Resilience

### Never Celebrate
Competition · Rankings · Comparison · Best/Top designations · Performance labels

---

## Illustration Principles

**Style inspiration**: Pixar · Disney Adventure Books · Khan Kids  
**Characters**: Diverse — gender, region, appearance. Indian settings and context where appropriate.  
**Expressions**: Curious · Optimistic · Confident

**Forbidden illustration subjects**: Corporate workers · Office scenes · Stock business people · Enterprise graphics · Corporate icon packs as illustration substitutes

---

## Design Review Gate (12 gates — all must pass)

Before approving any screen:

| # | Gate | Question |
|---|---|---|
| 1 | Hero | Does the child feel like the hero of this screen? |
| 2 | Companion | Is a companion present where appropriate for this module? |
| 3 | Visual Storytelling | Does visual storytelling exist — illustration or journey element? |
| 4 | Mobile First | Is this mobile-first — tested at 375px? |
| 5 | No ERP | No ERP appearance — no tables, dense forms, corporate grey? |
| 6 | No Dashboard | No dashboard appearance — no KPI panels, no admin layout? |
| 7 | No Corporate | No corporate appearance — no stock imagery, enterprise patterns? |
| 8 | Curiosity | Does this create curiosity — wanting to explore more? |
| 9 | Hope | Does this feel hopeful in tone? |
| 10 | Parent Safe | Would a parent feel proud their child uses this? |
| 11 | School OK | Looks like a life skills tool, not a game? |
| 12 | Accessible | WCAG AA, 44px touch targets, prefers-reduced-motion respected? |

If any gate fails → **Redesign.**

---

## Experience Tests

### Child Experience Test
Would a child think:
- "This is my adventure."
- "I want to explore."
- "This feels fun."
- "I want to come back."

If not → Redesign.

### Parent Trust Test
Would a parent think: **Safe · Healthy · Constructive · Positive · Trustworthy**  
If not → Redesign.

### School Acceptance Test
Would a school think: **Professional · Life Skills Focused · Future Ready · Character Building · Educational**  
If not → Redesign.
